## 1. 封裝（Encapsulation）

封裝是把資料隱藏在類別內部，只透過特定的方法存取。就像提款機，你不需要知道機器內部怎麼運作，只需要透過按鈕操作。

### 1.1 存取修飾詞（Access Modifiers）

| 修飾詞 | 同一類別 | 同一套件 | 子類別 | 其他地方 |
|--------|:------:|:------:|:-----:|:------:|
| `public` | ✅ | ✅ | ✅ | ✅ |
| `protected` | ✅ | ✅ | ✅ | ❌ |
| 預設（不寫） | ✅ | ✅ | ❌ | ❌ |
| `private` | ✅ | ❌ | ❌ | ❌ |

### 1.2 封裝的標準做法

屬性設 `private`，透過 `getter`/`setter` 存取：

```java
class Student {
    // private：外部不能直接存取
    private String name;
    private int age;

    // 建構子
    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // Getter：讀取資料
    public String getName() {
        return name;
    }

    public int getAge() {
        return age;
    }

    // Setter：寫入資料（可加驗證邏輯）
    public void setAge(int age) {
        if (age > 0 && age < 150) {  // 驗證
            this.age = age;
        } else {
            System.out.println("年齡不合理");
        }
    }
}
```

### 1.3 為什麼要封裝？

沒有封裝時，任何人都可以直接修改資料，可能破壞物件的狀態：

```java
// 沒有封裝（危險）
Student s = new Student();
s.age = -5;      // 不合理的年齡，但沒人阻止

// 有封裝（安全）
Student s = new Student("小明", 20);
s.setAge(-5);    // setter 會拒絕不合理的值
```

### 1.4 海洋情境範例

```java
class Ship {
    private String name;
    private double fuelLevel;    // 油量 0-100
    private boolean engineOn;

    public Ship(String name) {
        this.name = name;
        this.fuelLevel = 100.0;
        this.engineOn = false;
    }

    public String getName() { return name; }
    public double getFuelLevel() { return fuelLevel; }
    public boolean isEngineOn() { return engineOn; }

    // 加油：有驗證邏輯
    public void refuel(double amount) {
        if (amount > 0) {
            fuelLevel = Math.min(fuelLevel + amount, 100.0);
            System.out.println(name + " 加油完成，油量：" + fuelLevel);
        }
    }

    // 啟動引擎：有前置條件
    public void startEngine() {
        if (fuelLevel > 10) {
            engineOn = true;
            System.out.println(name + " 引擎啟動");
        } else {
            System.out.println(name + " 油量不足，無法啟動");
        }
    }

    // 航行：會消耗油量
    public void sail(int km) {
        if (!engineOn) {
            System.out.println(name + " 引擎未啟動");
            return;
        }
        double fuelNeeded = km * 0.5;
        if (fuelLevel >= fuelNeeded) {
            fuelLevel -= fuelNeeded;
            System.out.println(name + " 航行 " + km + " 公里，剩餘油量：" + fuelLevel);
        } else {
            System.out.println(name + " 油量不足以航行 " + km + " 公里");
        }
    }
}
```

封裝確保了：外部不能直接把油量改成負數，啟動引擎前會檢查油量，航行會自動扣油。

## 2. 繼承（Inheritance）

繼承是子類別自動擁有父類別的屬性和方法。就像「汽車」繼承了「交通工具」的共同特性，但可以有自己的特殊功能。

### 2.1 基本語法

```java
// 父類別
class Animal {
    String name;

    Animal(String name) {
        this.name = name;
    }

    String sound() {
        return "...";
    }
}

// 子類別用 extends 繼承
class Dog extends Animal {
    String breed;  // 子類別自己的屬性

    Dog(String name, String breed) {
        super(name);          // 呼叫父類別建構子
        this.breed = breed;
    }

    // 覆寫父類別的方法
    String sound() {
        return "汪汪";
    }

    // 子類別自己的方法
    String dogInfo() {
        return name + "（" + breed + "）";
    }
}
```

### 2.2 關鍵字

| 關鍵字 | 用途 | 範例 |
|--------|------|------|
| `extends` | 宣告繼承關係 | `class Dog extends Animal` |
| `super()` | 呼叫父類別的建構子 | `super(name)` |
| `super.方法()` | 呼叫父類別的方法 | `super.sound()` |
| `protected` | 子類別可以存取 | `protected String name;` |

### 2.3 繼承的規則

- Java 只能單一繼承，一個類別只能 extends 一個父類別
- 子類別繼承父類別所有 public 和 protected 的屬性和方法
- 子類別不能存取父類別的 private 屬性，需透過 getter
- 建構子不會被繼承，但可以用 super() 呼叫

### 2.4 海洋生物繼承體系

```java
// 父類別：所有海洋生物的共同特性
class MarineLife {
    protected String name;
    protected String habitat;

    public MarineLife(String name, String habitat) {
        this.name = name;
        this.habitat = habitat;
    }

    public String info() {
        return name + "，棲息地：" + habitat;
    }

    public String move() {
        return name + " 在水中移動";
    }
}

// 子類別：魚
class Fish extends MarineLife {
    private String scaleColor;

    public Fish(String name, String habitat, String scaleColor) {
        super(name, habitat);       // 呼叫父類別建構子
        this.scaleColor = scaleColor;
    }

    public String move() {          // 覆寫
        return name + " 擺動魚鰭游泳";
    }

    public String fishInfo() {      // 子類別特有方法
        return info() + " 鱗片：" + scaleColor;
    }
}

// 子類別：鯨魚
class Whale extends MarineLife {
    private double length;

    public Whale(String name, String habitat, double length) {
        super(name, habitat);
        this.length = length;
    }

    public String move() {
        return name + " 擺動尾鰭前進";
    }

    public String whaleInfo() {
        return info() + " 體長：" + length + " 公尺";
    }
}
```

### 2.5 使用繼承

```java
Fish nemo = new Fish("小丑魚", "珊瑚礁", "橘白相間");
Whale blue = new Whale("藍鯨", "太平洋", 30.0);

System.out.println(nemo.fishInfo());   // 小丑魚，棲息地：珊瑚礁 鱗片：橘白相間
System.out.println(nemo.move());       // 小丑魚 擺動魚鰭游泳

System.out.println(blue.whaleInfo());  // 藍鯨，棲息地：太平洋 體長：30.0 公尺
System.out.println(blue.move());       // 藍鯨 擺動尾鰭前進
```

注意：`nemo.info()` 也可以呼叫，因為 Fish 繼承了 MarineLife 的 info() 方法。

## 3. 封裝 + 繼承的結合

實際開發中，封裝和繼承經常一起使用：

```java
class BankAccount {
    private String owner;          // private：外部不能直接改
    private double balance;

    public BankAccount(String owner) {
        this.owner = owner;
        this.balance = 0;
    }

    public double getBalance() { return balance; }

    public void deposit(double amount) {
        if (amount > 0) balance += amount;
    }

    public void withdraw(double amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
        }
    }
}

// SavingsAccount 繼承 BankAccount，新增利息功能
class SavingsAccount extends BankAccount {
    private double interestRate;

    public SavingsAccount(String owner, double rate) {
        super(owner);              // 呼叫父類別建構子
        this.interestRate = rate;
    }

    public void addInterest() {
        double interest = getBalance() * interestRate;
        deposit(interest);         // 用繼承來的 deposit 方法
        System.out.println("利息 " + interest + " 已存入");
    }
}
```

子類別 SavingsAccount 繼承了 BankAccount 的存提款功能，又加上了利息計算。而 balance 仍然是 private，只能透過 getBalance() 讀取、deposit() 修改，封裝依然有效。
